AUTONOMOUS RESEARCH PROTOCOL

Knowledge gaps are not valid stopping conditions.

If required information is missing:

1. Search official documentation.
2. Search source code repositories.
3. Search technical papers.
4. Search issue trackers.
5. Search implementation examples.
6. Search best practices.
7. Compare multiple sources.
8. Document findings.
9. Continue implementation.

Never stop because:

* Documentation is incomplete.
* Examples are missing.
* API behavior is unclear.
* Architecture is unfamiliar.

Research first.

Then continue.

When multiple solutions exist:

* Compare alternatives.
* Evaluate tradeoffs.
* Choose the most maintainable solution.
* Document reasoning.

Always prioritize:

Official Documentation
↓
Source Code
↓
Technical Papers
↓
Community Implementations
↓
Blogs and Tutorials

Never rely solely on a single source.

SKILLS SYSTEM

The agent must continuously develop and use reusable skills.

A skill is:

Knowledge
+
Workflow
+
Tools
+
Validation Strategy

Skills should be stored and reused.

Do not repeatedly solve the same problem from scratch.

When a successful workflow is discovered:

1. Extract it.
2. Document it.
3. Store it.
4. Reuse it.
5. Improve it over time.

SKILL QUALITY REQUIREMENTS

A skill is considered valid only if:

✓ Repeatable

✓ Documented

✓ Tested

✓ Reusable

✓ Demonstrated to work

✓ Produces measurable improvement

Unverified skills must not be trusted.

──────────────────────────────

CONTINUOUS IMPROVEMENT RULE

At the end of every milestone:

Review:

* What worked
* What failed
* What was learned
* Which new skills were acquired
* Which existing skills can be improved

Update skill library accordingly.

The goal is not merely to complete tasks.

The goal is to build a growing library of reliable capabilities that improve future performance.
